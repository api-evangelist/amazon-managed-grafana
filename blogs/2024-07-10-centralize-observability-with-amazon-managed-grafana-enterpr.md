<p>Observability is a critical aspect for maintaining the health and performance of any distributed system. Organizations rely on data from diverse sources, including AWS services as well as third-party ISVs (independent software vendor) to gain insights into their system’s health. Establishing secure connections to these diverse data sources enables visualization and analysis of observability data in a single pane of glass view, driving data-driven decision making.</p> 
<p><a href="https://aws.amazon.com/grafana/">Amazon Managed Grafana</a> is a fully managed service for Grafana, a popular open-source analytics platform that enables you to query, visualize, and alert on data from a wide variety of data sources. Grafana offers an extensible architecture, with plugins that can be used to enhance user experience with a wide range of functionalities such as querying data from external data sources for better insights and improved visualizations. <a href="https://docs.aws.amazon.com/grafana/latest/userguide/grafana-plugins.html">Enterprise plugins</a> are exclusive data source plugins, built and maintained by <a href="https://grafana.com/">Grafana Labs</a>, connecting to proprietary tools or platforms, such as Splunk, Dynatrace, or Databricks. Enterprise plugins provide simplified user experiences, enabling you to query these systems alongside AWS services within a Grafana dashboard. For example, you could correlate your <a href="https://aws.amazon.com/eks/">Amazon Elastic Kubernetes Service</a> (Amazon EKS) performance metrics from A<a href="https://aws.amazon.com/prometheus/">mazon Managed Service for Prometheus</a>, application logs and traces from A<a href="https://aws.amazon.com/opensearch-service/">mazon OpenSearch Service</a> with your incidents in ServiceNow for effective troubleshooting.</p> 
<p>With Amazon Managed Grafana Enterprise plugins, you can <a href="https://aws.amazon.com/about-aws/whats-new/2024/03/amazon-managed-grafana-enterprise-plugins-upgrade/">upgrade</a> your Amazon Managed Grafana workspace to access third-party enterprise data source plugins, directly from the Amazon Managed Grafana console, <a href="https://aws.amazon.com/cli/">AWS Command Line Interface</a> (AWS CLI), or <a href="https://aws.amazon.com/developer/tools/">AWS SDK</a>. An Amazon Managed Grafana Enterprise plugins license costs an additional $45 per active user per workspace, and this gives you access to support and on-demand training directly from Grafana Labs, in addition to all the Enterprise plugins listed in <a href="https://docs.aws.amazon.com/grafana/latest/userguide/upgrade-to-enterprise-plugins.html">Enterprise data sources available with Amazon Managed Grafana Enterprise plugins</a>. With the <a href="https://aws.amazon.com/grafana/pricing/">pay-as-you-go</a> pricing model,&nbsp;Enterprise plugins upgrade have no upfront fees or minimum commitments, you will pay only for what you use, based on an active user per workspace. You can easily identify the monthly usage and costs of using Enterprise plugins with the <a href="https://docs.aws.amazon.com/awsaccountbilling/latest/aboutv2/useconsolidatedbilling-procedure.html">consolidated billing report</a>.</p>

<h3>Access Enterprise plugins</h3> 
<p>You can enable access to Enterprise plugins from the Amazon Managed Grafana console. When you create a new workspace, by default it does not have access to Enterprise plugins however, you can upgrade at any time. To enable Enterprise plugins in your Grafana workspace, refer <a href="https://docs.aws.amazon.com/grafana/latest/userguide/upgrade-to-enterprise-plugins.html#AMG-workspace-manage-enterprise">Managing your access to Amazon Managed Grafana Enterprise plugins</a>.</p> 
<p>With Enterprise Plugins upgrade, you get access to Grafana Labs consulting and support services. To enable this, your AWS account must be linked with a Grafana Labs account token. Linking consists of getting a token from a Grafana Labs account that is used in Amazon Managed Grafana to register the account. To create a Grafana token, refer <a href="https://docs.aws.amazon.com/grafana/latest/userguide/upgrade-to-enterprise-plugins.html#AMG-workspace-register-enterprise">Link your account with Grafana Labs</a>.</p>

<p>To automate your Enterprise plugins management, you can also use <a href="https://docs.aws.amazon.com/cli/latest/reference/grafana/associate-license.html">AWS CLI</a>, or <a href="https://docs.aws.amazon.com/grafana/latest/APIReference/API_AssociateLicense.html">Amazon Managed Grafana API</a>. You must use `ENTERPRISE` for the `licenseType`, and pass in a valid Grafana Labs token for the `grafana-token`.</p> 
<p><strong>Using Amazon Managed Grafana API</strong></p> 
<pre><code class="lang-bash">POST /workspaces/&lt;workspace-id&gt;/licenses/ENTERPRISE?Grafana-Token=&lt;grafana-token&gt; HTTP/1.1</code></pre> 
<p><strong>Using AWS CLI</strong></p> 
<pre><code class="lang-bash">aws grafana associate-license \
--region &lt;region&gt; \
--workspace-id &lt;workspace-id&gt; \
--license-type ENTERPRISE \
--grafana-token &lt;grafana-token&gt;
</code></pre> 
<h3>Switch from AWS Marketplace Grafana Enterprise license to Amazon Managed Grafana Enterprise plugins</h3> 
<p>To switch from AWS Marketplace Grafana Enterprise license to Amazon Managed Grafana Enterprise plugins, simply cancel your subscription from AWS Marketplace and upgrade from your Amazon Managed Grafana console. Your data source configurations will stay as it is after you complete the upgrade in Amazon Managed Grafana console. For frequently asked questions, refer <a href="https://docs.aws.amazon.com/grafana/latest/userguide/upgrade-to-enterprise-plugins.html#AMG-ws-mp-license-faq">FAQ for AWS Marketplace Enterprise users</a>.</p> 
<h3>Data transfer cost when Enterprise plugins is enabled</h3> 
<p>Just like connecting to other AWS managed services in the same region, there is no data transfer cost when you connect, query, or alert on the enterprise data sources that are connected to your Amazon Managed Grafana workspace. However, if your enterprise data sources are required to be routed from an external network through your <a href="https://aws.amazon.com/vpc/">Amazon Virtual Private Cloud</a> (VPC) prior to the connection to the Amazon Managed Grafana workspace using the <a href="https://docs.aws.amazon.com/grafana/latest/userguide/AMG-configure-vpc.html">VPC connection configuration</a>, then standard data transfer costs will be applied between AWS and the external network. Data transfer cost for common architectures can be found in the blog <a href="https://aws.amazon.com/blogs/architecture/overview-of-data-transfer-costs-for-common-architectures/">Overview of Data Transfer Costs for Common Architectures</a>.</p>
